# Scholar-Wendao v0.4 框架迭代清单

> 基于 v0.3 首个验证案例(Bernard Stiegler 蒸馏)实战暴露的问题
>
> 调研时间:2026-05-05

## 实战发现的 15 个问题(按严重度排序)

### P0 — 蒸馏深度不足(根本问题)

**1. "本地优先模式"未强制读 PDF 全文**
- 当前:Phase 1 写"先读本地素材,按 7 维度归类"
- 实战:只读了 Obsidian Card 元数据(用户已整理的引文),**没有任何一部本地 PDF 全文进入 Phase 2 提炼**
- 后果:用户合理质疑"是不是没把所有著作都下载做蒸馏?"
- 修复:Phase 1 增加 `extract_pdf_evidence.py` 强制 step,自动抽取每部本地 PDF 的目录/序言/首尾关键章节到 `_pdf_evidence/{book}.md`,作为 Phase 2 必须输入

**2. 没有"已下载素材→蒸馏证据"的链路追踪**
- 当前:Phase 1 → Phase 2 是黑箱,用户不知道哪部 PDF 进入了哪个概念
- 修复:SKILL.md 每个核心概念末尾增加"证据来源"小节,明示来自 `_pdf_evidence/{book}_ch{n}.md`

### P1 — Agent 可靠性

**3. 7 个并行 agent 经常不写文件**
- 实战:Agent 1 写到错误路径(我手动搬移);Agent 2 退缩;Agent 3 只在消息里返回未真写
- 修复:agent prompt 末尾必须包含"返回前 `cat` 出最终文件路径";主 agent 在 Phase 1.5 前必须 `ls` 验证文件存在,缺则兜底自写

**4. biography-protocol 的"狱中读哲学"BRACKETING 没自动校验**
- 当前:子 agent prompt 里强调,主 SKILL.md 输出无强制校验
- 修复:`quality_check.py` 增加"narrative-bracketing 检测":对"形成性事件"段落检查是否同时出现"事实层"+"叙事层"标注

### P2 — 文献采集精度

**5. harvest_works.py 消歧策略隐式**
- 当前:取第一候选(本次 Stiegler 恰好正确,但是 hack)
- 修复:显式按 works_count 排序选最大;接近的多候选 prompt 用户确认

**6. OpenAlex 误归(coauthor / 序作者作收)**
- 实战:"On the Existence of Digital Objects" 是 Yuk Hui 的书,因 Stiegler 撰序而被归到 Stiegler
- 修复:增加 `is_first_author` / `author_position` 字段过滤

**7. OA 下载实际成功率 < 25%**
- 实战:42 部 OA 中只 12 部成功;其余是 doi.org 重定向 / Cairn 落地页 / OpenEdition 登录墙 / DOAJ 摘要页
- 修复:(a) 跳过 doi.org 改用 unpaywall API;(b) 对落地页用 HTML 解析;(c) 对失败项写明落地页 URL 供手动跟踪

**8. annas_acquire.py 在受限网络下挂死**
- 修复:(a) `--manifest-only` flag 不实际下载只生成清单;(b) host 端 wrapper 脚本

### P3 — 与用户既有归档冲突

**9. download_open_access.sh 强制 fr/en/zh 子目录**
- 实战:用户的 Library 是扁平命名 `Stiegler{year}.pdf`,脚本造成污染且沙箱无权清理
- 修复:`_library_config.md` 中 `archive_layout: flat / by-language` 指定;命名 template `Stiegler{year}_{slug}[_{lang}].pdf`

**10. "本地优先模式"判断条件模糊**
- 实战:用户"在 Library 有一些"介于无和全有之间,无清晰处理路径
- 修复:增加"扫描+决策"step:扫 `$SCHOLAR_WENDAO_LIBRARY/{slug}/` 计算覆盖率(已有 / harvest 主要专著数),自动选模式 + 报告

### P4 — 质量检查脆弱

**11. quality_check.py 静态检查 regex 太脆弱**
- 实战:`r"调研时间[:：]\s*\d{4}"` 在用户写 `**调研时间**:2026` 时检测失败(粗体打断正则)— 第一次跑只得 90/100
- 修复:regex 改为 `r"调研时间[*\s]*[:：][*\s]*\d{4}"`

**12. 漫画化检测的"≤10/千字"是弱文字约束**
- 实战:LLM 自己作为执行者也超阈值(20/千字)
- 修复:增加输出后处理 step——每段输出末尾 LLM 自评术语计数,超阈值自动重写;或者挂 `density_check(text)` callable 工具

### P5 — 模板/结构

**13. 自杀/死亡边界缺失**
- 当前:humble-epistemics.md 的 5 项不包含"死亡-尊重-边界"
- 实战:我为 Stiegler 案例手动加了"额外警示"作为第 6 项
- 修复:作为 humble-epistemics 第 6 项,默认强制写入

**14. Phase 6 软链方向反了**
- 当前 prompt:`ln -sfn ~/.claude/skills/X examples/X`(假设 canonical 在 .claude)
- 实战:沙箱无法访问 ~/.claude/,实际更合理的是 canonical 在 git 仓库内
- 修复:框架文档明确 `~/.claude/skills/{X}` 是**软链目标**而非 source-of-truth

**15. examples/ 体积管控缺失**
- 实战:stiegler-perspective 596KB,主要是 07-archive.json 315KB
- 修复:`.gitattributes` 把 `*-archive.json` 标 LFS;或大 json 改压缩格式;或 archive.json 默认不进 git 只留 archive.md

## 建议的 v0.4 优先级

**应做(P1)**:3, 4 — agent 可靠性影响每个新蒸馏案例
