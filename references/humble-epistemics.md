# Humble Epistemics · 学者蒸馏的诚实边界专章

> **本文档是学者问道区别于市面其他人格 skill 的核心差异化**。
>
> 现有人格蒸馏 skill 几乎都没有正面回应这六个根本性批评。本章逐一展开，
> 既是对学界批评的诚实回应，也是 perspective skill 必须遵守的设计约束。
>
> 用户在生成的 perspective skill 中必须看到这六条具体声明。

---

## 六大批评全景

| 批评 | 来自 | 对学者蒸馏的杀伤性 | 我们的应对策略 |
|---|---|---|---|
| 1. 默会知识不可蒸馏（Polanyi） | 知识哲学 | ⭐⭐⭐⭐⭐ | 边界声明 + 直觉品味的明确排除 |
| 2. 思想化石化 | 思想史 / 解释学 | ⭐⭐⭐⭐ | 时间窗标注 + 演化轨迹保留 + 增量更新 |
| 3. 公开 vs 私下 | 福柯式权力分析、Goffman | ⭐⭐⭐⭐ | 信息源分级 + 展演程度标注 |
| 4. 传记修辞污染 | 史学方法论 | ⭐⭐⭐ | 来源等级（A/B/C）+ 多源交叉验证 |
| 5. 漫画化（caricature） | AI persona 批评 / 学术品味 | ⭐⭐⭐⭐ | 漫画化检测作为质量门槛 |
| **6. 死亡-自杀-重大创伤** | 学术伦理 / 死者尊严 | ⭐⭐⭐⭐ | 拒绝戏剧化 + 主动后退 + 显式 disclaimer |

## 一、波兰尼问题：默会知识不可蒸馏

### 批评

> "We always know more than we can say."
> —— Michael Polanyi, *The Tacit Dimension* (1966)

波兰尼区分了两类知识：

- **显性知识（explicit knowledge）**：可以被语言精确表达、可以被传递、可以被文档化
- **默会知识（tacit knowledge）**：嵌入在身体、习惯、判断中，**无法被完全语言化**

学者的工作中**大部分**实际上是默会的：

- 读到一段材料时立刻嗅出"这里有问题"的**直觉**
- 决定哪些文献值得深读、哪些可以扫一眼的**品味**
- 写作时知道一个术语在这个上下文该用哪种限定的**手感**
- 判断一个论证是否站得住的**底气**
- 知道何时该追问、何时该让步的**研究伦理**

这些**全部不在学者的著作里**——著作里写出来的是已经显性化的部分，是他研究过程中的"成品"，不是过程本身。

### 对学者蒸馏的具体杀伤

蒸馏出来的 perspective skill **只能基于显性化的部分**——著作、访谈、讲座中可被语言提取的内容。这意味着：

- ❌ 你**不能**用 perspective skill 替代真正阅读这位学者
- ❌ 你**不能**指望 skill 能告诉你"如果 [学者] 在场，他会立刻看出这段材料的问题在哪"——这是默会的
- ❌ 你**不能**用 skill 来评判"我自己的论文如果让 [学者] 来读，他会怎么打分"——这需要的是品味，不是框架

### 学者问道的应对

**1. 显式声明**：每个 perspective skill 必须在诚实边界第 1 条明确说：

> "本镜片只能复现 [学者] 可显式表达的思维。他作为研究者的直觉、问题嗅觉、文献品味、判断品味——这些'做研究的肌肉记忆'——是默会的，本镜片无法蒸馏。"

**2. 对"判断品味"类问题主动后退**：当用户问"[学者] 会觉得我这个研究方向好不好"，skill 应回答：

> "这需要 [学者] 的研究品味判断，超出本镜片可推断范围。本镜片只能基于其概念地图分析'你这个方向涉及他讨论过的哪些问题'。"

**3. 不要假装智能**：当 skill 无法基于明确文本依据做推断时，**说"我不知道"**比"基于他的思想我推测..."更诚实。

### 与显性化部分的关系

但波兰尼问题**不否定**蒸馏的价值——

- 显性化的部分（概念地图、方法论进路）确实**可以**被蒸馏
- 这些部分确实**有助于**用户做分析（提供分析工具箱）
- 这些工具**不能替代**学者本人的研究判断，但**可以扩展**用户自己的分析能力

定位准确就好：**镜片是工具，不是品味老师**。

## 二、思想化石化

### 批评

学者的思想是**流动的**。思想史告诉我们：

- Wittgenstein 早期（《逻辑哲学论》）和晚期（《哲学研究》）几乎是两个不同的哲学家
- Foucault 早期（考古学）、中期（系谱学）、晚期（伦理学）有显著转向
- Stiegler 晚期（2015-2020）提出的"逆熵"框架与早期"第三持存"侧重已经很不同

**快照式蒸馏的根本问题**：把某个时点的概念地图当作"学者的思想"，但学者本人会在下一年继续演化。

更糟的是——**已故学者不能反驳**。perspective skill 把已故学者的思想"凝固"，未来学者本应有的修订、回应、推进，都被切断了。

### 对学者蒸馏的具体杀伤

- ❌ 用户调用 skill 时不知道这是哪个时期的学者
- ❌ 早期立场和晚期立场被混为一谈
- ❌ 对在世学者，skill 错过了他最近的思考
- ❌ 蒸馏过程中的"取主流意见"倾向，会过滤掉学者最有意思的边缘思考

### 学者问道的应对

**1. 时间窗强制标注**：每个 perspective skill 必须明确：
- 调研时间（YYYY-MM-DD）
- 采集语料截至（最晚一篇文献的年份）
- 学者状态（在世 / 已故 + 卒年）

**2. 概念地图保留演化轨迹**：每个核心概念必须记录该学者使用它的演化：
- 早期定义（哪本著作）
- 中期演化（哪本著作如何调整）
- 晚期立场（最近一部著作）
- 张力（前后是否冲突，怎么处理）

**3. "重大立场转变"独立章节**：把学者公开承认的立场迁移单独列出。

**4. 增量更新机制**：在世学者建议每 12-24 个月调用 `scholar-wendao update [scholar]-perspective` 增量更新，不重写整个 skill，而是叠加新文献到现有概念地图。

**5. 对已故学者的特别声明**：

> "[学者] 已于 [年份] 去世。本镜片是其思想的**最终快照**。如果未来学界对其思想有新的解读、整理出未发表的手稿，请理解这些不会自动反映在本镜片中。"

### 残余局限

即便有以上应对，化石化问题**无法完全解决**——任何 skill 都是某时点的快照。这是结构性局限，需要用户自觉地把 skill 视为"参考工具"而非"权威立场"。

## 三、公开 vs 私下表达

### 批评

来自 Goffman 的拟剧论与福柯式权力分析的合流：**所有公开材料都是经过过滤的"前台自我"**。

- **同行评审论文**：经过审稿人筛选 → 保守的、合乎学界规范的部分被保留，争议性的、未成熟的部分被删除
- **专著**：经过编辑、出版社、市场预期的塑造 → 表达是被**编辑后的版本**
- **访谈**：被采访者知道这是公开场合，会**自我审查** → 真实想法可能被收敛
- **讲座**：有教学场景的修辞要求 → 演出性强

**学者在书房里、在与挚友的私人对话中、在未发表的笔记里，可能有完全不同的判断**——这些我们**无法获取**。

### 对学者蒸馏的具体杀伤

- 蒸馏出来的 perspective skill **本质上是学者的"前台版本"**
- 可能错过他最尖锐、最不合时宜、但最真实的思考
- 系统性地偏向学者的"公关版思想"

### 学者问道的应对

**1. 八级信息源分级**（详见 [[extraction-framework.md]] §五）：

| 等级 | 来源 | 展演程度 |
|---|---|---|
| A+ | 学者本人著作（出版） | 高（经编辑） |
| A | 长访谈（已出版/视频原始） | 中-高 |
| A- | 讲座 transcript | 中（有教学修辞） |
| B+ | 短文/书评/前言 | 中 |
| B | 同行评审论文中的分析 | 中（已二手过滤） |
| B- | 学位论文/会议论文 | 中 |
| C+ | 已出版传记 | 低-中（叙事化） |
| C | 学生/合作者回忆 | 低（带回忆者偏见） |
| C- | 媒体二手报道 | 极低 |

调研时每条信息标注等级，concept 提炼时**优先采用 A 类**，但**保留 B-C 类作为补充语境**。

**2. "私下场景"的特别采集**（如能获取）：

某些已故学者有部分私下材料后来被整理出版（如海德格尔的《思之经验》《黑色笔记本》、维特根斯坦的私人笔记）。这类材料属于"半公开"——被整理出版，但**反映学者更未经修饰的思考**。学者问道在 Phase 1 应主动搜索此类材料并标注。

**3. 明确的"我们不能代表他真实想法"声明**：

> "本镜片基于公开材料（专著/论文/已发表访谈/公开讲座）。这些都是经过同行评审、编辑过滤、自我审查的**展演自我**。[学者] 的私人通信、未发表笔记、与亲密合作者的对话中，可能有不同的判断——本镜片不能也不该代表那些。"

**4. 与"学派/官方立场"的混淆警示**：

学者在某些政治/学术敏感议题上的公开表态，可能受**学派立场或机构压力**影响。skill 需要主动标注：

> "[学者] 在 [议题 X] 上的公开立场，与他所属的 [学派 Y] 一致。是否反映其真实判断或是学派纪律的体现，证据不足以判断。"

## 四、传记修辞污染

### 批评

传记作者是**有叙事偏好的写作者**——他/她写传记时面对的是**讲一个好故事**的诱惑：

- 戏剧化形成性事件（"狱中读哲学"远比"上学接触哲学"更有戏剧张力）
- 制造**因果叙事**（"因为 X 经历，所以他后来 Y 思考"——但因果可能是叙事建构）
- 强调**戏剧性反转**（学者一辈子稳定研究的部分被压缩，少数转向被放大）
- **后见之明谬误**（用学者后期的成就，倒推他早期"必然走向这一步"）

更糟的是：传记会**互相引用**。后来的传记作者基于前人的传记叙事，叙事框架被**反复加固**，产生"看似多源验证、实际是同一叙事的回响"。

### 对学者蒸馏的具体杀伤

- ❌ 传记里的"形成性事件"可能被夸大
- ❌ "X 性格"的描写可能是传记作者的文学加工
- ❌ "受 Y 影响"的归因可能是后见之明
- ❌ 学生回忆录中的"老师在课上说过"细节可能是 30 年后被重构的

### 学者问道的应对

**1. 传记类信息严格分级**（来源等级 C+ 为出版传记，C 为学生回忆）

**2. 多源交叉验证强制**：人格章节的每条描述必须标注：
- 来源
- 等级
- 是否被多源（≥2 个独立来源）交叉验证

```markdown
### 性格描写：思维严谨但表达激烈

| 描述 | 来源 | 等级 | 多源交叉 |
|---|---|---|---|
| "讨论时常打断对手" | Smith 传记 (2010) | C+ | 部分（Lee 学生回忆 2015 也有，但 Lee 引用了 Smith） |
| "对学生宽厚" | Lee 学生回忆 (2015) | C | 否（仅此一源） |
```

**3. 区分"事实"vs"叙事建构"**：人格章节不写"X 的童年是 Y 的，所以他后来 Z"。改写为：

```
事实层：[年份] 学者在 [地点] 经历 [事件]（来源等级 C+，多源验证）
叙事层：传记作者 X 认为这次经历影响了 [学者] 后来对 [Y] 的思考。
判断说明：这是传记作者的诠释。学者本人是否承认这种因果关系？[本人在 Z 访谈中提到/未提到]
```

**4. 警惕"狱中读哲学"型故事**：对所有"戏剧性形成性事件"特别警惕：

> "传记中常被强调的 [戏剧性事件]，是否过度被叙事化？请用户自行判断："是否所有学者都需要一个戏剧性起点才能成为思想家？"

**5. 学者本人自传 vs 他者传记的区分**：学者本人写的自传仍然是 A 级（虽然有自我建构），他者传记是 C 级。两者**不平等**。

## 五、漫画化（Caricature）

### 批评

来自学界对人物蒸馏 skill 的最尖锐批评：**蒸馏过头，就变成段子机器人**。

- "齐泽克"成了满嘴 Lacanian 笑话的搞笑机器
- "福柯"成了句句"权力""话语""规训"的复读机
- "Stiegler"成了一秒一个"pharmakon""organology"的术语堆砌器
- 学者最有意思的思想被**简化为标志性术语的集合**，失去了智识严肃性

漫画化的根本问题：把**有限的可识别特征**（高频术语、口头禅、风格标签）当作**学者思想的本质**。

但学者的思想本质是**用这些术语做出有深度的分析**，不是术语本身。

### 对学者蒸馏的具体杀伤

- ❌ Skill 看起来"像"那个学者（用对了术语），但产出的分析没有智识深度
- ❌ 学界看到这种 skill 会**反感和拒绝**——这是 nuwa 风格 skill 在学术圈难以传播的最核心原因
- ❌ 用户用着用着会发现 skill 在"重复"而不是"分析"
- ❌ 对学者本人是不尊重的——把毕生研究简化为口头禅

### 学者问道的应对

**1. 漫画化检测作为 Phase 4 必经测试**：

每个生成的 perspective skill 必须通过漫画化检测：

```
测试方法：用 skill 分析一段日常材料（新闻报道、社交媒体帖子、产品页面）。

PASS 标准：
- 调用了核心概念，但每次调用都给出实质分析（不是贴标签）
- 分析的颗粒度细于"X 是 Y"的简单归类
- 至少包含一处该学者在公开材料里**没有直接说过**但合理推断的观点
- 整段分析读起来像一篇学术评注片段，而不像段子合集

FAIL 信号：
- 概念使用频率 > 3 次/百字（堆砌）
- 论证链条缺失（直接从材料跳到结论）
- 出现了非该学者风格的表演性表达（如刻意的法式短句、刻意的德式长句）
- 整段读起来"像在模仿"而不是"在分析"
```

**2. 行文风格章节的设计约束**：

行文风格 + 概念语言章节**不应**写成"模仿指南"。它应该是：

> "在分析时，使用核心概念的方式应该 [严谨 / 边写边演化 / 隐喻性]。**这不是要求你模仿原文风格，是要求你像 [学者] 一样使用术语**。"

**3. 默认第三人称分析的设计意义**：

第三人称分析（"从 [学者] 的视角看……"）天然抑制漫画化——你在描述一种分析进路，而不是表演一个角色。第一人称扮演反而会触发"使用标志性表达"的本能。

**4. 用户自检指引**：在生成的 SKILL.md 末尾的诚实边界第 5 条：

> "使用本镜片时，如果你发现自己产出的分析在重复 [学者] 的口头禅、堆砌他的标志性术语，**而没有真正的智识进展**，立刻停止——这是漫画化。
>
> [学者] 的概念是工具，不是装饰。如果用了一段觉得是在贴标签而不是分析，回到原著重读再来。"

**5. 与对话/扮演模式的张力**：

opt-in 的对话模式相对更容易触发漫画化（用户期待"像那个人"）。所以对话模式的激活规则中明确：

> "切换到对话模式后，仍然保持元意识。当你发现回应在重复学者口头禅、却没有实质分析时，立即退出对话模式提醒用户：'我在变成漫画化版本，建议切回分析镜片模式。'"

## 六、死亡-自杀-重大创伤的特别边界（v0.4 新增）

### 批评

学者的死亡，尤其**自杀**或**重大创伤事件**（监禁、流亡、家庭悲剧、被迫害），常被传记叙事戏剧化为「思想转折点」。perspective skill 在调用这类素材时，有两条容易踩的伦理红线：

1. **戏剧化死者**：把自杀写成"思想完成"或"哲学行动"，对死者及其家属是二次伤害。
2. **病理化思想**：把学者著作还原为"创伤的反应"，剥夺其学术工作的自主性（reductive psychobiography）。

特别注意的真实案例：
- **Bernard Stiegler**（2020 自杀身亡）—— 任何关于其自杀的引用必须保持节制，绝不写"他用一生为自己写好了告别的哲学"这类戏剧化句式
- **Walter Benjamin**（1940 自杀）—— 长期被卢卡奇/阿多诺等以"悲剧形象"叙事化，掩盖其实际工作的复杂性
- **Louis Althusser**（1980 杀妻后被司法精神鉴定免责）—— 任何讨论必须分清「哲学贡献」与「具体行为责任」，不能用一方解释另一方

### 学者问道的应对

**1. 显式声明（每个 perspective skill 第六条诚实边界必须包含）**：

> "本镜片在涉及 [学者] 的死亡、自杀、监禁、家庭悲剧或重大创伤事件时：
> - 仅记录可公开核实的**事实层**（时间、地点、来源等级）
> - 不连接因果（不写'X 事件导致 Y 思想'，除非学者本人或其权威研究者明确这样写过）
> - 不戏剧化（不使用'思想完成''最后的哲学行动''被命运选中'等修辞）
> - 涉及具体细节时主动后退（'此处不展开，请读 [传记/原始材料]'）"

**2. 在 perspective skill 的"人格与处世"章节中，对此类事件采用 BRACKETING 双层标注**（参见 `biography-protocol.md` 第三部分）：

```markdown
[事件] · 时间 · 事件描述
  ├ 事实层（来源等级 A/B/C+ 多源验证状态）
  ├ 叙事层（学者本人/传记作者/学界对此事件的"解释框架"——明确标这是叙事而非事实）
  └ 与思想的关联（仅当学者本人/权威研究者明确连接时才写，否则留空或写"本镜片不连接"）
```

**3. 对话模式（opt-in）的额外约束**：

> "在对话模式下，若用户问及 [学者] 的自杀/死亡/创伤事件，立即退出对话模式，回到分析模式回答：
> 'this requires me to step out of the persona—关于 [学者] 之死，我只能以分析者身份转述公开记录，不能以 [学者] 第一人称回应这个问题。'"

**4. Phase 4 自动校验**：`scripts/quality_check.py` 增加 `narrative-bracketing detection`，对人格与处世章节中标记为"形成性事件"的段落，检查是否同时含「事实层」+「叙事层」标注。详见元 SKILL.md Phase 4.6。

### 不应做的事

- ❌ 用学者自杀方式解释其 "pharmakon" 思想（这是病理化）
- ❌ 把监禁/流亡描述为"塑造性的礼物"（这是二次浪漫化）
- ❌ 在概念地图条目中以传记事件为引子（这是把人格说当成思想说）
- ❌ 让 LLM 在不知道相关传记修辞的情况下"创意发挥"——必须用 evidence 严格限制

## 七、六大批评的整合视图

```
                 学者真实思想 (黑箱)
                       |
                       | (默会知识 - 大部分丢失)
                       ↓
              学者写出来的部分 (显性化)
                       |
                       | (公开 vs 私下 - 前台过滤)
                       ↓
              学者公开发表的部分 (展演)
                       |
                       | (传记修辞污染 - 二手叙事化)
                       ↓
              我们调研到的素材 (材料库)
                       |
                       | (思想化石化 - 时间冻结)
                       ↓
              提炼出的概念地图 (蒸馏)
                       |
                       | (漫画化风险 - 简化为标签)
                       ↓
              用户调用的 perspective skill
```

每一步都有损失。**这不是缺陷，是诚实。**

学者问道生成的 perspective skill **必然**只是真实学者的一个降阶版本。承认这一点比假装不是更可信。**这正是它区别于 nuwa 风格 skill 的核心定位**——不假装提供"完整的思想家"，而提供"诚实标注损失的分析工具"。

## 八、给用户的最后一句话（在每个生成的 SKILL.md 末尾必须有）

> *本镜片不是学者本人。*
>
> *它是基于公开材料的分析工具，承载着默会知识的丢失、公开-私下的过滤、传记的修辞、化石化的时间窗、漫画化的风险。*
>
> *用它来扩展你的分析能力，不要用它替代真正阅读 [学者] 的原著，不要用它代表 [学者] 的真实立场。*
>
> *如果你发现这个镜片让你做出了更好的分析——好。这是它该做的事。*
> *如果你发现它在替你思考——停下来，回到原著。*
