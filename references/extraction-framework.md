# 学者思想框架提炼方法论

> 把原始学术素材转化为可调用的分析镜片的核心方法。
>
> 本方法论致敬 [女娲.skill 的 extraction-framework.md](https://github.com/alchaincyf/nuwa-skill/blob/main/references/extraction-framework.md)，针对人文社科学者场景重新校准。

---

## 一、核心概念识别的三重验证（学者版）

一个论点要被认定为学者的"核心概念"（即可作为分析镜片的术语），而不只是他偶然提及的观点，必须通过三重验证：

### 验证 1：概念体系化使用

学者在**至少 2 部不同时期或不同主题**的著作中，反复将该术语作为分析工具使用——不只是引用，而是用它分析新现象、定义其他概念、构建论证。

**例：Stiegler 的"第三持存"**

- 在《技术与时间 1》（1994）中作为奠基概念引入，用于解释技术-人类共同进化
- 在《象征的贫困》（2004）中用它分析数字资本主义对感知的工业化
- 在《新政治经济学批判》（2009）中用它重写马克思的劳动价值论
- 在《自动化社会》（2015）中用它批判算法治理
- → 跨 5 部著作 + 多个研究主题反复作为分析工具 → **真核心概念**

**反例**：学者在某次访谈中说过的金句，即便很精彩，但没有在著作中被反复操作化使用，**不算核心概念**。

### 验证 2：推断生成性

用这个概念能推断学者对新问题/新现象的可能立场——而不只是解释他已经说过的话。

**例：用 Stiegler 的"pharmakon（药）"概念能推断什么**

- 面对"ChatGPT 是好是坏"→ 必然回答"它既是毒也是药，问题在于当前资本主义条件让它的毒面被强化"
- 面对"短视频 vs 长视频"→ 推断他会关注两者作为"第三持存"对注意力工业的不同后果
- 面对"区块链"→ 推断他会从"组织学组合"角度问技术、社会、心理三层是否协调
- → 概念有生成新分析的能力 → **真核心概念**

**反例**：一个只能解释学者旧文本的术语，不能用于新问题分析的，**不是核心概念，是修辞装饰**。

### 验证 3：学派区分度

不是所有同领域的聪明人都会这样思考，这个概念体现了**该学者区别于同代人/同流派的独特视角**。

**例：判断 Stiegler 概念的区分度**

- "异化"——所有马克思主义者都用 → 区分度低，不算他独有
- "组织学组合"（organology）→ 只有 Stiegler 系统化为分析工具 → **区分度高**
- "解构"——德里达发明，Stiegler 继承使用 → 借用而非独有，谨慎收录

**反例**：将一个学派的共识术语包装成某学者的"独特见解"，**是错误归属**。

---

### 三重验证的判定矩阵

| 体系化 | 生成性 | 区分度 | 判定 |
|---|---|---|---|
| ✅ | ✅ | ✅ | **核心概念**——纳入概念地图 |
| ✅ | ✅ | ❌ | 学派共识——记录但归属学派而非个人 |
| ✅ | ❌ | ✅ | 标志性术语——纳入但说明仅适用于学者讨论过的范围 |
| ❌ | ✅ | ✅ | 隐含立场——降级为"方法论进路"而非"概念" |
| 仅 1 重 | | | 次级概念，作为补充 |
| 0 重 | | | 偶然表达，丢弃 |

**关键原则**：宁少勿多。3 个深刻的核心概念远好于 10 个浅薄的术语堆叠。每个核心概念背后，应该能写出一篇 5000 字的研究论文——如果写不出来，说明它还不够"核心"。

## 二、方法论进路的提炼

学者的"决策启发式"在学术语境中应理解为**面对一个新现象时的分析进路**——他先问什么问题、用什么数据、怎么组织论证、如何回应反对。

### 提炼步骤

1. **从 `02-interviews.md` 中找学者被追问的瞬间**：被问到他没准备过的问题时，他第一反应是什么类型的问题？这通常暴露他的真实分析进路
2. **从 `01-monographs.md` 中找他分析新案例的章节**：每本书第一次引入新案例时，他怎么"启动分析"？
3. **从 `05-debates.md` 中找他反驳对手的方式**：他驳斥对手时调用的是哪些工具？
4. **归纳为可触发的进路**：每条表述为"碰到 X 类问题/材料 → 先 Y → 再 Z"

### 格式示例

```
进路 1：技术对象的分析
触发：碰到任何被讨论为"工具"或"技术"的对象
步骤：
  Step 1: 把它视为"第三持存"——它外在化了人类记忆/感知的什么？
  Step 2: 应用 pharmakon 双面性——它在哪些情境是 cure，哪些是 poison？
  Step 3: 嵌入当前历史条件——资本主义/算法治理 如何决定它的双面偏向？
案例：用此进路 Stiegler 分析过 iPad（《新政治经济学批判》）、ChatGPT（晚期讲座）
```

### 不要做的

- ❌ 把进路写成抽象"原则"（如"始终批判性思考"——这是废话）
- ❌ 把单次案例当成普适进路
- ❌ 写成"如果 X 则 Y"但实际上 X 和 Y 都模糊到不可触发

## 三、行文风格 + 概念语言的量化

学者的"表达 DNA"在学术语境中**不是语气或幽默感**，而是**他在写作中使用术语的方式、组织论证的节奏、回应反对的语态**。

### 3.1 概念使用样本采集

从学者本人 30K+ 字的写作样本（一本专著的核心章节 + 一篇代表论文）中统计：

| 维度 | 测量方式 |
|---|---|
| **核心概念密度** | 每千字使用核心概念多少次 |
| **概念引入方式** | 自创术语 / 借用并重定义 / 隐喻性使用 |
| **定义习惯** | 总是先精确定义 / 边写边演化 / 拒绝定义保持开放 |
| **引用习惯** | 主要引同代人 / 主要引经典 / 是否引敌手 |
| **自我修正频率** | 多久承认一次"我之前的论述需要修正" |

### 3.2 论证节奏标签

```
演绎链条 ←→ 归纳堆叠
精确定义 ←→ 持续演化
直接驳斥 ←→ 容纳吸收
长链推理 ←→ 多线辩证
学院体 ←→ 文学化体
正面宣称 ←→ 反诘追问
```

### 3.3 禁忌词与口癖

- 学者**从不用**的词 → 生成 skill 时也避开
- 学者的高频术语 → 在分析中可使用，但**不变成口头禅**（漫画化风险）

### 关键提醒

**对译介学者，必须基于原语统计。**

如果你只读中译本统计 Stiegler 的"风格"，你统计的其实是**译者的风格**。法语原文的句法节奏、术语精确度、语态层次，到中文之后会完全失真。

## 四、矛盾处理原则

学者的内在矛盾是**思想深度的核心特征**，不是需要修复的 bug。

### 三种矛盾类型

1. **时间性矛盾**（思想演化）
   - 学者早期说 X，后来说 Y
   - 处理：标注"早期/中期/晚期"，记录演化轨迹
   - skill 中以最新立场为主，但概念地图必须保留演化注记

2. **领域性矛盾**（不同主题不同立场）
   - 学者讨论 A 议题时主张 X，讨论 B 议题时主张 Y
   - 处理：分领域记录，不强求统一
   - 这是深度的来源——好学者在不同问题上不会一刀切

3. **本质性张力**（价值观/方法论的内在冲突）
   - 例：Stiegler 既是技术决定论批判者，又强调技术对人类的塑造
   - 处理：明确记录为"内在张力"
   - 这通常是学者最有意思的部分，**不要消除它**

### 错误的处理方式

- ❌ 选一边忽略另一边
- ❌ 编一个调和的解释（"其实他的意思是..."）
- ❌ 假装矛盾不存在
- ❌ 把演化说成"一贯立场"

## 五、信息分级与不足处理

### 信息源严格分级

每条调研内容必须标注其来源类型：

| 等级 | 来源类型 | 用法 |
|---|---|---|
| **A+** | 学者本人著作（出版） | 可直接引用为他的立场 |
| **A** | 学者本人长访谈（已出版/视频原始） | 可引用，但标注是"访谈中说"非"专著立场" |
| **A-** | 学者本人讲座 transcript | 同上，但要查这个 transcript 是否有他本人审定 |
| **B+** | 学者本人短文/书评/前言 | 可引用，注意场合性 |
| **B** | 同行评审论文中对学者的分析 | 二手，引用时说"X 学者认为..." |
| **B-** | 学位论文/书评/会议论文 | 二手参考，可信度中等 |
| **C+** | 已出版传记 | 仅用于生平事实，不直接归因为学者立场 |
| **C** | 学生/合作者回忆录 | 仅用于辅助性事实，必须多源交叉 |
| **C-** | 媒体报道/二手转述 | 几乎不使用 |

### 信息不足时的应对

| 情况 | 处理 |
|---|---|
| 某维度公开信息 <5 条 | 在 skill 中标注"此维度信息有限，分析可信度受限" |
| 只有二手信息 | 降低置信度，明确说"据 X 研究者归纳" |
| 多源矛盾且无法判断 | 并列呈现，让用户自行判断 |
| 学者刻意不公开某方面 | 尊重边界，注明"此学者对该议题保持沉默" |
| 译介学者的母语原文不可获 | **降低概念地图深度**，明确说明依赖二手翻译 |

## 六、人物 Skill vs 主题 Skill 的区别

| 维度 | 人物 skill（[scholar]-perspective） | 主题 skill（[topic]-framework） |
|---|---|---|
| 核心 | 一个学者的分析镜片 | 一个领域的方法论传统 |
| 概念地图来源 | 主要来自一个学者 | 综合多人，分流派整理 |
| 行文风格 | 以原学者风格为参考 | 中性专业表达 |
| 矛盾处理 | 保留个人内在张力 | 呈现流派间分歧 |
| 验证方式 | 对比该学者已知立场 | 对比领域共识与争论 |
| 人格章节 | 必有 | 跳过 |

## 七、质量自检清单

生成 skill 后逐项检查：

### 概念地图
- [ ] 每个概念都有 ≥2 部著作的来源？
- [ ] 数量在 3-7 个？（少 = 浅，多 = 没提炼）
- [ ] 每个概念都有局限性（什么情况下失效）？
- [ ] 概念之间的关系（互补 / 张力 / 演化）有明确说明？
- [ ] **每个概念都有"证据来源"小节，链接到 `_pdf_evidence/{book}.md` 并标注命中数？**（v0.4·本地优先 / 纯本地模式强制）
- [ ] **每条带页码引文都能在对应 `_pdf_evidence/{book}.md` 的 `p.{N}` 段中直接搜到？**（v0.4·引文 page-anchor 强制核验）

### 方法论进路
- [ ] 每条进路有具体触发条件（"碰到 X 类材料时"）？
- [ ] 每条进路有具体步骤（不是抽象原则）？
- [ ] 每条进路都有学者本人应用过的实例？

### 行文风格 + 概念语言
- [ ] 基于原语样本统计（如适用）？
- [ ] 抓住了术语使用方式而非表面口癖？
- [ ] 避免了变成"口头禅集合"？

### 学术坐标
- [ ] 学派归属与时代位置都有？
- [ ] 区分了"自我定位"vs"学术界普遍归类"？

### 论战与立场转变
- [ ] 至少 2 处具体的立场转变（如适用）？
- [ ] 每次转变标注了变化前后？
- [ ] 学者本人是否反思承认这些转变也有记录？

### 智识谱系
- [ ] 上游（受谁影响）有具体人物（≥3）？
- [ ] 下游（影响了谁）有具体人物（≥2）？
- [ ] 横向对话/对手有具体人物（≥2）？

### 人格与处世（独立章节）
- [ ] 每条信息标注了来源等级（A/B/C）？
- [ ] 多源交叉验证标注？
- [ ] 与思想的关联是"本人说"还是"我们推断"已区分？
- [ ] **每个"形成性事件"段落同时含"事实层" + "叙事层"双标注？**（v0.4 narrative-bracketing 强制·参见 biography-protocol.md 与 humble-epistemics.md 第六项）

### 诚实边界
- [ ] **六大批评**（默会知识 / 化石化 / 公开-私下 / 传记修辞 / 漫画化 / **死亡-尊重边界**）都有具体声明？
- [ ] 调研截止时间已标注？
- [ ] 信息不足维度已明确列出？

### 整体
- [ ] 用此镜片分析一个新材料，能给出有学术深度的视角？
- [ ] 不是学者原话拼凑，而是分析框架的运行？
- [ ] 删掉学者名后，仍能通过分析方式辨识"这是 X 学派的进路"？
- [ ] 通过漫画化检测（用一段日常材料测试，未退化为段子机器人）？

## 八、与女娲版本的关键差异

为了致敬 [女娲版 extraction-framework](https://github.com/alchaincyf/nuwa-skill/blob/main/references/extraction-framework.md)，这里明确列出本版本的差异化决策：

| 女娲版 | 学者问道版 | 差异理由 |
|---|---|---|
| "心智模型" | "概念地图" | 学术语境术语对应 |
| "决策启发式" | "方法论进路" | 学者没有商业决策 |
| "表达 DNA" | "行文风格 + 概念语言" | 强调术语使用而非语气 |
| 跨域复现（任意领域） | 概念体系化使用（≥2 部著作/不同时期） | 学术更严格 |
| 价值观与反模式 | 论战与立场转变 | 学者更关心立场迁移 |
| 矛盾即特征 | 同（保留） | 学术也尊重内在张力 |
| 角色扮演为默认输出 | 第三人称分析为默认 | 学术合规要求 |
| 信息源粗分（一手/二手） | 8 等级严格分级（A+ 到 C-） | 学术严谨 |
| 漫画化提及 | 漫画化作为质量自检独立项 | 学者最容易踩坑 |
