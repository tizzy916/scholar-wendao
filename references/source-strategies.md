## 二、按学科的特化策略

### 哲学

**核心档案**：
- 国际哲学会议论文集 / Stanford Encyclopedia of Philosophy
- PhilPapers（元数据 + OA）
- 学者所属哲学协会的 newsletter（如 SPEP for continental philosophy）

**特殊处理**：
- 哲学家**必读其本人最完整的著作版本**（如 Heidegger Gesamtausgabe），不能只读译本
- 哲学家的"思想转向"特别重要，必须在 Phase 2.6 立场转变中重点处理
- 二手哲学研究文献量极大，调研时**先搜 Cambridge Companion / Routledge Companion / Stanford Encyclopedia 的某学者条目**，作为二手综述的入口

### 社会学

**核心档案**：
- ISA（国际社会学学会）期刊
- 学者所属国家的社会学协会 archive
- 民族志类研究的 fieldnotes（部分被出版）

**特殊处理**：
- 社会学家的方法论文献（《什么是社会学方法》《社会学的想象力》等）需独立处理为方法论进路提取的核心材料
- 实证型社会学家：要采集其代表性的实证研究（数据、田野），不只是理论著作
- 社会学家的合作者网络比哲学家重要——同一研究计划下的合作者论文要扫一遍

### 文化研究

**核心档案**：
- *Cultural Studies* 期刊及其学派分支期刊
- 学者参与的文化研究中心档案（如 Birmingham CCCS、Goldsmiths）
- 学者关于流行文化/媒体的具体分析文本

**特殊处理**：
- 文化研究者经常做"具体文本分析"——这些分析本身就是方法论的展示，必须采集
- 文化研究往往与政治介入紧密结合，立场转变与政治背景关联强，§5 论战章节要重点处理

### 历史学

**核心档案**：
- 学者所在档案馆 / 大学档案
- 历史学协会 archive
- 学位论文（历史学家的博士论文常常是其方法论的奠基）

**特殊处理**：
- 历史学家的"史料处理方式"是其核心方法——这超越概念地图，需要在 §方法论进路 中详细记录其史料类型偏好与处理流程
- 历史学家的争论常关于"史实诠释"，§5 论战章节的处理要细致

### 政治理论 / 法理学

**核心档案**：
- *Political Theory*、*Constellations*、*Law & Social Inquiry* 等期刊
- 学者参与的政策报告 / 立法建议
- 学者公开发表的时事评论（这些常是其理论的应用展示）

**特殊处理**：
- 政治理论家的"规范立场"与"分析方法"要区分清楚
- 学者作为公共知识分子的言论与作为学者的言论要分级处理

## 三、按学者特征的策略调整

### 已故学者

- 必查**遗作集 / 全集 / 标准版**（Gesamtausgabe / Œuvres complètes / 全集）
- 必查**未发表手稿、笔记**（如 Wittgenstein Nachlass、Foucault 法兰西学院讲座、Heidegger 黑色笔记本）
- 学生/合作者的**追思文集**（特别是学者去世后 5 年内出版的）
- 注意**学派后续整理者**的解读偏向（如 Lacan 学派对其文本的整理）

### 在世学者

- 学者本人**主页 / 学术档案**最优先
- **最近 2-3 年的出版物**特别注意（化石化风险）
- 推特 / X / Bluesky / Bsky 学术活动（如有，作为补充）
- **学者本人维护的视频频道**（YouTube 讲座 / 个人 podcast）

### 译介学者（用户母语非学者母语）

- **必须采集原语版本**——译本只作参考
- 原语 → 英译 → 中译，**信息保真度逐级下降**
- 概念地图 §2.1 中的概念名必须包含原语
- §2.4 行文风格必须基于原语样本统计
- 如无法获取原语，明确在 SKILL.md 诚实边界声明"本镜片基于英译/中译本，对学者真实行文风格的还原能力受限"

### 多语言学者（如学者用多种语言写作）

- 例：Stiegler 主要用法语，但晚年用英语演讲；Habermas 用德语写作但在英语世界被广泛译介
- 各语言著作都要采集，**不能假设某种语言代表全部**
- 学者在不同语言场景中的立场是否一致 → 在 §2.5 论战章节中处理

## 四、Agent 启动时的策略组装

Phase 1 启动 7 个 agent 时，主 agent 需要根据学者特征**动态组装**策略：

```python
# 伪代码：策略组装逻辑
def assemble_source_strategy(scholar):
    strategies = []

    # 主语言策略
    primary_lang = scholar.primary_language
    strategies.append(LANG_STRATEGIES[primary_lang])

    # 学科策略
    discipline = scholar.discipline  # philosophy / sociology / cultural_studies / history / political_theory
    strategies.append(DISCIPLINE_STRATEGIES[discipline])

    # 状态策略
    if scholar.deceased:
        strategies.append(DECEASED_STRATEGY)
    else:
        strategies.append(LIVING_STRATEGY)

    # 译介策略
    if scholar.primary_language != user_native_lang:
        strategies.append(TRANSLATION_STRATEGY)

    # 中国学者特别策略
    if scholar.country == "China":
        strategies.append(CHINA_BLACKLIST)  # 强排除知乎、公众号、百度百科

    return merge_strategies(strategies)
```

主 agent 把组装后的策略写入 `references/research/_strategy.md`，作为 7 个 sub-agent 的共享指引。

## 五、信息源不足时的应对升级

| 情况 | 升级策略 |
|---|---|
| 学者主语言资源 <10 条 | 扩大到次要语言（英译、第二母语） |
| 二手研究 <5 篇 | 扩大到学位论文、会议论文集 |
| 访谈资源稀缺 | 寻找学者参与的圆桌讨论 / 对谈集 |
| 完全是冷门学者 | Phase 0.5 时就告知用户"质量会受限"，引导用户提供本地素材 |

## 六、信息源可信度的最后一道关：学者本人确认

理想情况下：

- 在世学者：通过其学术联系人或本人邮箱发送 perspective skill，请求 review（学术礼貌）
- 已故学者：通过其学派的代表性研究者 review

这一步在 v1 不强制要求，但**未来 v2 应该考虑**——让 perspective skill 成为学界共同维护的开放档案，而非单方面蒸馏的"作品"。

---

> *信息源决定了 perspective skill 的质量上限。*
> *垃圾进，垃圾出——但严格的源策略，能把进的东西尽可能干净。*
