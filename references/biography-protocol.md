# Biography Protocol · 生平人格采集与处理规范

> 学者问道相比 nuwa 的差异化点之一：**生平人格作为独立章节，但严格分级标注信息源**。
>
> 本文档是 Phase 1 Agent 6（生平传记 + 智识谱系）和 Phase 2.7（人格与处世章节）的操作规范。

---

## 一、为什么生平对学者比对企业家更重要

| 维度 | 企业家蒸馏（女娲风格） | 学者蒸馏（学者问道） |
|---|---|---|
| 思想-生平耦合度 | 弱（产品决策与个人成长相对解耦） | **强**（学者的研究问题常源于生平经历） |
| 形成性事件的重要性 | 商业起点 | 学术起点 + 思想转向触发 |
| 人格特质的相关性 | 中（影响管理风格） | **高**（影响研究方法、学派归属、写作风格） |
| 信息可获取性 | 低（企业家私人生活保护） | **中-高**（学者经常出版传记、自传、回忆录） |

**举例：Stiegler 的"狱中读哲学"**

- 1978 年武装抢劫被捕
- 服刑 5 年期间开始系统读哲学
- 出狱后师从 Derrida
- 这段经历**直接塑造了**他关于"技术外在化"的核心思想——他自述：在监狱里，他亲身体验了"被剥夺时间感"是什么意思，这成为他理解技术-时间-人的起点

**忽略生平**：你只能蒸馏出他的概念地图（"第三持存""pharmakon"）但不知道这些概念的**情感重量**。
**整合生平**：你能理解为什么他对"时间被剥夺"如此敏感，为什么"pharmakon"必须包含暴力的可能。

## 二、生平信息的来源等级（重申，参见 [[extraction-framework.md]]）

| 等级 | 来源 | 用途 | 限制 |
|---|---|---|---|
| **A+** | 学者本人**专著中的传记性段落** | 可作为传记事实 | 仍是自我叙事，但属公开记录 |
| **A** | 学者本人**自传 / 回忆录** | 可作为传记事实 + 学者自述 | 自我建构强 |
| **A-** | 学者本人**长访谈中谈自己** | 可作为辅助传记事实 | 公开场合的自我呈现 |
| **B+** | 学者**私人通信**（已出版） | 揭示更接近私下的判断 | 已被编辑选择 |
| **B** | **终身合作者 / 配偶 / 子女回忆** | 第一手私人观察 | 情感倾向重 |
| **B-** | **学生回忆录** | 师生场景的观察 | 师生关系不对等可能扭曲 |
| **C+** | **专业传记作家撰写的传记** | 综合性叙事 | 文学化处理严重 |
| **C** | **学术对手的描述 / 论战中提及** | 反面观察 | 可能贬损 |
| **C-** | **媒体报道 / 维基百科** | 基础事实校对 | 不作为核心来源 |

## 三、采集流程

### Step 1: 基础事实清单

Agent 6 首先采集**可被多源验证的基础事实**：

```markdown
## 基础生平事实
- 出生：[年份] · [地点]
- 父母 / 家庭背景：[简述]（来源等级，多源验证）
- 教育：[学校年份] → [导师] → [学位时间地点]
- 主要任职：[机构、年份、职位]
- 重要学术里程碑：[首部专著、获奖、终身教职等]
- 主要居住 / 工作地（影响其学术圈）
- 去世（如适用）：[年份] · [地点] · [原因（如已知且学者本人/家属公开过）]
```

**验证标准**：基础事实至少需要 ≥2 个独立来源交叉验证才记录。如只有单源（如仅一本传记），明确标注"单源未验证"。

### Step 2: 形成性事件采集

寻找学者**自己反复提及**或**多份传记反复强调**的关键事件：

```markdown
## 形成性事件 X：[事件名]

### 事实层
- 时间：[年份]
- 事件：[简述]
- 来源等级：[A / A- / B / C+]
- 多源交叉：[是 / 否 / 部分]

### 学者本人的解读（来源 A 类）
- 学者在 [何处] 提到此事件 → 简述
- 他将此事件与 [思想/概念] 关联 → 简述

### 他者的解读（来源 B/C 类）
- 传记作者 [姓名] 在 [传记书名] 中如何叙述 → 简述
- 学者与他者解读的差异 → 标注

### 谨慎标注
- 这是"戏剧化形成性事件"型故事吗？（如戏剧性反转、监狱经历、突然顿悟）→ 警惕传记修辞
- 学者本人的因果叙事是否被后见之明影响？
```

### Step 3: 性格描写采集

性格描写**最容易被传记修辞污染**。采集时必须严格：

```markdown
## 性格特征 Y：[一句描述]

| 描述细节 | 来源 | 等级 | 多源交叉 |
|---|---|---|---|
| "[具体行为/反应描述]" | [传记/回忆录] | [B/C] | [是/否] |
| "[另一描述]" | [另一来源] | ... | ... |

### 学者本人的自我描述（如有）
- [访谈/自传中如何谈自己] → 简述

### 反面观察（如有）
- 学术对手 / 反对者描述 → 简述
- 与"主流叙事"的差异 → 标注

### 我们的判断
- 多源是否真的独立验证？还是后来的传记引用了前面的传记？
- 是否有"圣人化"或"妖魔化"倾向？
```

### Step 4: 重要他者关系

学者的研究方向常受**师承、合作、敌对**关系塑造。采集：

```markdown
## 师承

### 直接导师 [姓名]
- 关系起止：[年份]
- 学术影响：[继承了什么思想/方法]
- 与导师的张力：[是否有立场分歧、决裂、修正]
- 来源：[文献]

### 远程谱系（精神导师）
- [人物 1] → 影响维度
- [人物 2] → 影响维度

## 合作者
- [核心合作者 1] → 合作的项目 / 共著作品 / 关系性质
- ...

## 学术对手 / 论战对象
- [对手 1] → 论战议题 / 时间窗 / 是否和解
```

### Step 5: 重大争议与处世

学者的**公开争议**比简单的人格描写更有信息量——因为它涉及他在压力下的真实判断：

```markdown
## 重大争议 X：[简述]

- 时间：[年份]
- 起因：[何事]
- 学者的回应：
  - 公开声明：[简述 + 出处]
  - 行为表现：[简述 + 来源]
- 他者的解读：[支持者 / 反对者各自的叙述]
- 这次争议暴露的处世原则：[谨慎归纳，标注是"我们的推断"]
- 后续：[争议如何收场 / 学者是否反思]
```

## 四、写入 `biography/` 目录的格式

```
references/biography/
├── timeline.md            # 结构化生平时间线（按年份）
├── personality.md         # 性格特征（来源分级）
├── relations.md           # 重要他者关系
└── controversies.md       # 重大争议与处世
```

每个文件的开头都必须有**信息源声明**：

```markdown
# [文件名]

## 信息源声明

本文档采集时间：YYYY-MM-DD
主要传记类来源：
- A 级（本人自述）：[列举]
- B 级（亲属/合作者/学生回忆）：[列举]
- C 级（传记作家 / 媒体）：[列举]

**警惕**：本文档涉及生平人格描写，**所有内容均带有不同程度的叙事建构**。
请理解这只是**用于辅助理解学者思想的语境**，**不是关于学者本人的事实陈述**。

---
```

## 五、写入最终 SKILL.md 的"人格与处世"章节

参见 [[scholar-template.md]] 的对应章节。关键约束：

### 1. 必须分级标注

每条描述必须显示来源等级（A/B/C），不能"洗"成一句平直的陈述。

### 2. 区分"事实"vs"叙事建构"

错误示范：
> "Stiegler 因为狱中读哲学，所以发展出了对技术外在化的关注。"

正确示范：
> "Stiegler 1978 年因武装抢劫服刑（事实层，来源 A：本人自述 + C+：多本传记），服刑期间开始系统读哲学（事实层，来源 A）。
>
> 他本人在 2003 年访谈中将此经历与他后来对'技术外在化'的关注关联起来（来源 A）。
>
> 多本传记作家强化了这种因果叙事（来源 C+），但也有研究者（如 Daniel Ross）指出，他在狱中接触的具体哲学家（Husserl、Heidegger）才是更直接的思想塑造者，'监狱经历'本身可能被叙事化（来源 B：研究文献）。
>
> **判断**：'狱中读哲学'是 Stiegler 思想形成的重要语境之一，但应避免线性因果叙事。"

### 3. 警惕"圣人化"与"妖魔化"

很多传记会把学者描述成"严谨""慷慨""孤独""特立独行"——这些是**传记标准模板**。一旦发现描写过于符合"伟大学者"叙事模板，**降低可信度**，标注"叙事建构嫌疑"。

### 4. 私人议题的克制

学者的：
- 婚姻 / 家庭 / 子女
- 性取向 / 健康 / 心理状态
- 政治倾向（除非是其学术工作的核心）

→ **除非学者本人公开讨论过 + 与其学术思想直接相关 + 多源验证**，否则**不写入**。

学者问道**不窥探**学者隐私。哪怕是已故学者。

## 六、特殊场景

### 蒸馏导师 / 师兄 / 同事（私人学者）

- 学者问道**无法**从公开渠道蒸馏私人学者
- 用户必须提供素材：导师的论文 / 讲义 / 面谈录音 transcript / 给学生的反馈邮件
- 生平人格章节**几乎完全跳过或极简**——只记录用户能直接观察到的、与学术思想相关的部分
- **绝对不**根据用户对导师的描述生成"导师人格画像"——这超出 perspective skill 的范围，进入个人关系的领域

### 蒸馏在世学者 + 隐私边界

- 在世学者的生平人格章节**保持最低必要范围**
- 只记录：基础公开事实 + 已出版的自传/访谈中本人提到的部分
- **不**采集：朋友圈/微博等社交平台的私人内容
- 即便用户能找到，**不**写入

### 蒸馏有争议的学者（如有政治污点 / 学术不端指控）

- 必须**公平呈现**——既呈现学者的思想贡献，也呈现争议
- 争议章节单列，标注事实 + 各方立场 + 时间线
- **不洗白**也**不批判**——学者问道是分析工具，不是道德裁判
- 给用户提供完整信息，让用户自己判断如何使用这个 perspective skill

经典案例：海德格尔的纳粹问题、福柯的后期争议、阿尔都塞的家庭事件——必须呈现，不可回避。

## 七、与 Agent 6 的协作

Agent 6（生平传记 + 智识谱系）的任务流：

```
1. 检索基础生平事实（多源交叉验证）
   → 输出：timeline.md（基础事实层）

2. 检索学者本人的自传性材料
   → 优先：自传、回忆录、访谈中的传记段落、专著序言
   → 输出：timeline.md 增加 A 级条目

3. 检索亲属 / 合作者 / 学生回忆
   → 输出：personality.md + relations.md

4. 检索专业传记
   → 输出：timeline.md + personality.md 的 C 级条目

5. 检索重大争议
   → 输出：controversies.md

6. 多源交叉与谨慎判断
   → 标注每条信息的来源等级与多源验证状态
   → 标注"叙事建构嫌疑"

7. 生成"人格与处世"草稿
   → 写入 06-genealogy.md 的人格部分
   → 等待 Phase 2.7 整合
```

## 八、伦理底线

学者问道在生平人格采集上的**底线**：

1. **不窥探**未被学者本人公开的私人生活
2. **不夸大**戏剧性事件
3. **不拼接**孤立细节生成虚假人物画像
4. **不替学者**做未公开的判断
5. **不简化**复杂人物为"伟大学者"或"问题人物"的标签
6. **承认**所有传记类信息都带叙事建构

学者是人，不是工具材料。**学者问道蒸馏的是思想分析镜片，不是人本身**。

---

> *生平给思想以重量，但不能替代思想本身。*
> *呈现生平是为了帮助理解思想的语境，不是为了制造"思想家的故事"。*
